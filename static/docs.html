Documentación — Residencial Guaraní API

Descripción general

Esta API REST sirve la landing page de Residencial Guaraní e implementa un chatbot de FAQs basado en un árbol de decisión por palabras clave. También expone los datos de departamentos, servicios y respuestas del chatbot.

Endpoints

POST /api/chat

Envía un mensaje al chatbot. Retorna la respuesta más relevante del árbol de decisión.

Request

POST /api/chat
Content-Type: application/json

{ "message": "cuánto cuesta un departamento?" }

Response

{ "id": 4, "response": "Nuestros departamentos van desde Gs. 2.500.000/mes..." }
GET /api/respuestas

Lista todas las respuestas del árbol de decisión.

GET /api/respuestas

[
  { "id": 2, "categoria": "horarios", "texto": "Nuestro horario de atención..." },
  { "id": 3, "categoria": "ubicacion", "texto": "Estamos ubicados en..." },
  ...
]
GET /api/respuestas/{id}

Obtiene una respuesta específica por su ID.

GET /api/respuestas/4

{ "id": 4, "categoria": "precios", "texto": "Nuestros departamentos van desde Gs. 2.500.000/mes..." }
GET /api/departamentos

Lista todos los departamentos disponibles con precio, superficie y características.

GET /api/servicios

Lista los servicios del residencial (mantenimiento, seguridad, áreas comunes, etc.).

Cómo agregar una nueva respuesta

1. Agregar el texto en handlers/respuestas.go, dentro de RespuestaMap:

12: {ID: 12, Categoria: "parking", Texto: "El estacionamiento cuesta Gs. 300.000/mes adicional."},

2. Agregar las palabras clave en handlers/chat.go, dentro de decisionTree:

{Keywords: []string{"parking", "auto", "cochera"}, RespuestaID: 12},

Cómo funciona el backend

main.go — Router y servidor

Usa el router chi. Registra middlewares (logger, recoverer, CORS), monta todos los endpoints de la API, sirve la Swagger UI y finalmente sirve los archivos estáticos de static/ como fallback con http.FileServer. El servidor escucha en el puerto 8080.

r.Post("/api/chat",            handlers.PostChat)
r.Get("/api/departamentos",    handlers.GetDepartamentos)
r.Get("/api/servicios",        handlers.GetServicios)
r.Get("/api/respuestas",       handlers.GetRespuestas)
r.Get("/api/respuestas/{id}",  handlers.GetRespuestaByID)
r.Get("/swagger/*",            httpSwagger.Handler(...))
r.Get("/docs",                 → static/docs.html)
r.Handle("/*",                 http.FileServer("static/"))

models/models.go — Structs

Define todos los tipos de datos que circulan entre handlers y JSON:

Departamento   → datos de cada departamento (precio en PYG, superficie, características)
Servicio       → servicios del residencial (icono, nombre, descripción)
Respuesta      → una entrada del banco de respuestas del chatbot (id, categoria, texto)
FAQ            → nodo del árbol de decisión (keywords []string, RespuestaID int)
ChatRequest    → body del POST /api/chat { "message": "..." }
ChatResponse   → respuesta del chatbot { "id": 4, "response": "..." }

handlers/respuestas.go — Banco de respuestas

RespuestaMap es un map[int]Respuesta que centraliza todos los textos del chatbot. Los handlers GetRespuestas y GetRespuestaByID lo exponen vía API. El handler de chat también lo usa para resolver un ID a texto.

var RespuestaMap = map[int]models.Respuesta{
    0:  { fallback },
    1:  { saludo },
    2:  { horarios },
    ...
    99: { easter egg },
}

handlers/chat.go — Árbol de decisión

decisionTree es un slice de FAQ. Cada entrada tiene un conjunto de palabras clave y el ID de la respuesta que debe devolverse si hay coincidencia. El algoritmo:

1. Normalizar el mensaje: minúsculas + quitar acentos (golang.org/x/text)
2. Si contiene un saludo → responder con ID 1
3. Recorrer decisionTree, contar cuántas keywords coinciden con las palabras del mensaje
4. Elegir la entrada con mayor puntaje → usar su RespuestaID
5. Buscar RespuestaMap[id] y devolver { id, response }

handlers/departamentos.go y servicios.go

Datos hardcodeados en slices de structs. Cada handler simplemente serializa el slice a JSON con json.NewEncoder(w).Encode(...). En una versión futura estos datos podrían venir de una base de datos.

Frontend — static/js/main.js

Al cargar la página hace dos llamadas fetch() en paralelo a /api/servicios y /api/departamentos, luego crea los elementos HTML de las cards dinámicamente con document.createElement.

Frontend — static/js/chatbot.js

El widget de chat escucha el submit del formulario, hace un POST /api/chat con el mensaje del usuario, y agrega la respuesta recibida al historial de mensajes. Maneja el toggle del panel y muestra un mensaje de bienvenida al cargar.
